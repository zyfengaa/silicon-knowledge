# 学习路线图

本文件详细说明各模块的前置依赖、学习方式和预计时长。

---

## 总览

从零基础到能系统性分析硬件系统的路径：

```
01 数字逻辑
   └→ 02 ISA 与 CPU 基础
         ├→ 03 CPU 流水线 ─→ 04 CPU 微架构进阶
         └→ 05 存储层次 ─────────┐
                                  ├→ 06 GPU 架构 → 07 CUDA 编程
                                  │                   ↓
                                  └→ 08 NPU → 09 TPU → 10 其他加速器
                                                              ↓
                                              11 系统分析与性能工程
```

---

## 模块详情

### 01 — 数字逻辑基础

**前置依赖**：无
**预估时长**：1-2 周
**学习方式**：阅读笔记 → 运行 Python 模拟代码 → 完成练习题
**关键产出**：
- 能手工推导布尔表达式、化简逻辑电路
- 理解加法器/乘法器的基本实现
- 理解 IEEE 754 浮点数的位级表示

### 02 — 指令集与 CPU 基本组成

**前置依赖**：01 数字逻辑
**预估时长**：2-3 周
**学习方式**：以 RISC-V RV32I 为主线 → 阅读笔记 → 写汇编代码 → 模拟单周期 CPU
**关键产出**：
- 能阅读和手写简单的 RISC-V 汇编
- 能画出单周期 CPU 的数据通路图
- 理解 RISC vs CISC 的设计哲学差异

### 03 — CPU 流水线

**前置依赖**：02 ISA 与 CPU 基础
**预估时长**：2 周
**学习方式**：阅读笔记 → 运行流水线模拟器 → 分析冒险场景
**关键产出**：
- 能画 5 段流水线的结构图（IF/ID/EX/MEM/WB）
- 能识别三类冒险并说明解决方案
- 能计算不同场景下的 CPI

### 04 — CPU 微架构进阶

**前置依赖**：03 CPU 流水线
**预估时长**：2-3 周
**学习方式**：阅读笔记 → 运行分支预测模拟器 → 编译执行 SIMD 代码 → 分析 false sharing
**关键产出**：
- 理解乱序执行中的 ROB / 保留站 / 寄存器重命名
- 能用 SIMD intrinsics 编写向量化代码
- 能解释 false sharing 的发生条件和影响

### 05 — 存储层次

**前置依赖**：02 ISA 与 CPU 基础
**预估时长**：2-3 周
**学习方式**：阅读笔记 → 运行缓存模拟器 → 动手写缓存感知优化的代码
**关键产出**：
- 能解释缓存的三种组织方式（直接映射/组相联/全相联）
- 能用分块（tiling）优化矩阵乘的缓存利用率
- 理解 TLB 和多级页表的工作原理

### 06 — GPU 架构

**前置依赖**：03 CPU 流水线（需要理解并行概念）
**预估时长**：3-4 周
**学习方式**：阅读笔记 → 运行基础 CUDA 代码 → 分析 warp 行为
**关键产出**：
- 理解 SIMT 与 SIMD 的本质区别
- 能解释 warp divergence 对性能的影响
- 理解 GPU 内存层次和合并访问条件

### 07 — CUDA 编程与优化

**前置依赖**：06 GPU 架构
**预估时长**：2-3 周
**学习方式**：阅读笔记 → 编写 CUDA kernel → 使用 Nsight Compute 分析性能
**关键产出**：
- 能编写高性能的 CUDA 内核（矩阵乘、规约等）
- 能用 occupancy 计算指导 launch 配置
- 能使用 Nsight 工具分析 kernel 的瓶颈

### 08 — NPU 神经网络处理器

**前置依赖**：05 存储层次
**预估时长**：2-3 周
**学习方式**：阅读笔记 → 运行脉动阵列模拟器 → 阅读经典 NPU 论文
**关键产出**：
- 理解脉动阵列的工作原理和数据流策略
- 理解量化如何影响硬件设计
- 能分析不同 NPU 架构的优劣（Eyeriss / DianNao / ANE / 昇腾）

### 09 — Google TPU

**前置依赖**：08 NPU（理解脉动阵列）
**预估时长**：1-2 周
**学习方式**：阅读笔记 + TPU 系列论文 → 尝试 JAX on Colab TPU
**关键产出**：
- 能复述 TPU v1~v4 每一代的核心改进
- 理解 MXU / HBM / SparseCore 的作用
- 能对比 TPU 与 GPU 在 AI 训练中的优劣

### 10 — 其他加速器

**前置依赖**：08 NPU（参照物）
**预估时长**：1-2 周
**学习方式**：阅读笔记 → 选 1-2 个感兴趣的方向深入
**关键产出**：
- 了解 FPGA、Cerebras、Groq、Graphcore、SambaNova 等设计理念
- 理解近存计算和存算一体的动机

### 11 — 系统分析与性能工程

**前置依赖**：模块 01~10
**预估时长**：3-4 周
**学习方式**：阅读笔记 → 跑 Roofline 分析 → 对真实程序做性能建模
**关键产出**：
- 能用 Roofline 模型判断程序的瓶颈类型
- 能使用 perf / Nsight / TPU profiling 工具
- 能对大模型训练系统做端到端的硬件分析
