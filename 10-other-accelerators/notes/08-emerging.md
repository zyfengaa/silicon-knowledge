# 08 — 前沿方向：光计算、模拟计算与量子计算

> 除了前面讨论的成熟加速器，还有一些更具颠覆性的计算范式正在探索中。光计算、模拟计算和量子计算都有可能从根本上改变计算方式，但它们目前都处于非常早期的阶段。

---

## 1. 光计算 (Optical Computing)

### 1.1 基本原理

光计算使用光子替代电子进行计算，利用光的物理特性（干涉、衍射、偏振）执行运算。

#### 光学乘法

光计算最自然擅长的操作是**矩阵乘法**，可以通过 Mach-Zehnder Interferometer (MZI) 的网格实现：

```
光学矩阵乘法器:
输入光信号 (编码为不同的波长/相位)
    ↓
MZI 网格 (每个 MZI 实现 2×2 酉变换)
    ↓
通过 MZI 的相位调节器设置权重
    ↓
输出光信号强度 = 输入 × 权重矩阵
    ↓
光电探测器 → 电信号输出
```

每个 MZI 可以看作一个可调的 2×2 矩阵乘法单元，通过级联 MZI 可以实现任意西矩阵。

#### 优势

| 特性 | 电子计算 | 光计算 |
|------|---------|-------|
| 速度 | GHz 级 (电子传输受限) | THz 级 (光速) |
| 并行度 | 空间有限 (布线) | 天然并行 (波分复用) |
| 功耗 | 连线寄生电容高 | 无 RC 延迟，低功耗 |
| 矩阵乘法复杂度 | O(N²) (数字) | O(1) (模拟光学) |

### 1.2 Lightmatter

Lightmatter 是目前最知名的光计算创业公司之一，推出了 Passage 和 Mars 产品：

```
Lightmatter Mars:
- 光学 AI 推理加速器
- 利用光学干涉实现矩阵乘法
- 输入/输出仍使用光电转换
- 混合光-电架构

优势: 在高吞吐量场景下能效比优于 GPU
挑战: 精度有限 (模拟计算), 与数字系统集成困难
```

### 1.3 主要挑战

1. **精度**：模拟光的强度、相位受噪声影响，难以达到 FP32 精度
2. **集成度**：光子器件尺寸远大于晶体管
3. **光电转换**：每次计算需要 E/O (电→光) 和 O/E (光→电) 转换，开销大
4. **非线性**：光学难以高效实现非线性和激活函数
5. **温度敏感**：干涉器对温度变化敏感

## 2. 模拟计算 (Analog Computing)

### 2.1 基本概念

模拟计算直接将物理量（电压、电流、电荷）作为计算变量，而不是将其数字化后再计算。

#### 模拟矩阵乘法

最核心的技术是利用**交叉阵列（Crossbar Array）** 在模拟域完成矩阵乘法：

```
Crossbar Array 结构:
         V1    V2    V3
         │     │     │
  ───────■─────■─────■──────  (权重行)
  G11    R11   R12   R13        ↑ 电流输出 I1
  ───────■─────■─────■──────
  G21    R21   R22   R23        I2
  ───────■─────■─────■──────
         │     │     │
        G31   G32   G33
             ↓
不挥发存储器 (RRAM/PCM/NOR Flash) 的电导值存储权重

I_i = Σ V_j × G_ij   (KCL 定律)
```

### 2.2 Mythic

Mythic 公司使用 NOR Flash 阵列实现模拟存内计算：

```
Mythic Analog AI 处理器:
- NOR Flash 阵列存储权重 (电导值表示)
- 模拟矩阵乘：输入电压 → 权重阵列 → 电流求和
- 模数转换器 (ADC) 将结果转回数字域
- 片上数字电路处理非线性/激活函数

优势:
- 完全消除权重数据的搬运
- 极高能效 (相对数字电路 10-100×)
- O(1) 时间完成矩阵乘法

挑战:
- ADC/DAC 精度和功耗
- 非易失存储器的噪声和漂移
- 工艺偏差带来的精度损失
```

### 2.3 历史与现状

| 时代 | 代表 | 特点 |
|------|------|------|
| 1940-60s | 模拟计算机 | 用于微分方程求解，精度低 (1-5%) |
| 1970s | 数字计算取代模拟 | 数字精度和可重复性胜出 |
| 2010s | AI 计算需求 | 能效比需求重新点燃模拟计算兴趣 |
| 2020s | Mythic, analog AI chips | 混合信号，定点精度 (INT4/INT8) |

## 3. 量子计算 (Quantum Computing)

### 3.1 基本原理

量子计算利用量子力学原理进行计算。与经典计算的核心区别：

| 经典 | 量子 |
|------|------|
| 比特：0 或 1 | 量子比特：α|0⟩ + β|1⟩ (叠加态) |
| 操作：逻辑门 | 操作：酉变换 |
| 状态确定可观测 | 观测导致坍缩 |
| 可克隆 | 不可克隆 (No-cloning theorem) |

### 3.2 量子算法

只有少数算法被证明有量子优势：

| 算法 | 用途 | 经典复杂度 | 量子复杂度 | 加速比 |
|------|------|-----------|-----------|-------|
| Shor's | 大整数分解 | 亚指数 | 多项式 | 指数级 |
| Grover's | 无结构搜索 | O(N) | O(√N) | 平方级 |
| HHL | 线性方程组 | O(N) | O(log N) | 指数级 (稀疏) |
| VQE | 化学模拟 | 指数级 | 多项式 (近似) | 指数级 |
| QAOA | 组合优化 | 指数级 | 多项式 (近似) | 指数级 |

### 3.3 Shor 算法

Shor 算法可以在多项式时间内完成大整数分解——这是 RSA 加密的基础：

```
RSA-2048 (2048-bit 整数分解):
经典计算机: 需要 ~10^12 年 (假设摩尔定律继续)
量子计算机: 需要 ~10^8 个物理量子比特, ~1 天
```

### 3.4 NISQ 时代

当前处于 **NISQ (Noisy Intermediate-Scale Quantum)** 时代：

```
NISQ 特征:
- 50-1000 个物理量子比特
- 量子门错误率 ~0.1-1% (远超容错阈值 ~10^-6)
- 无法纠错 → 只能运行短电路
- 尚未展示明确的量子优势

主要量子硬件平台:
| 平台 | 代表机构 | 量子比特数 (2024) | 错误率 |
|------|---------|------------------|--------|
| 超导 | IBM, Google | 1000+ | ~0.1% |
| 离子阱 | IonQ, Quantinuum | 30-50 | ~0.01% |
| 中性原子 | QuEra, Atom Computing | 1000+ | ~1% |
| 光量子 | Xanadu, PsiQuantum | 数百 (压缩态) | 不同度量 |
| 自旋 (硅) | Intel, Diraq | 数十 | ~0.1% |
```

### 3.5 主要挑战

1. **量子比特质量**：退相干时间短，门错误率高
2. **纠错开销**：一个逻辑量子比特需要 1000+ 物理量子比特
3. **可扩展性**：从 100 到 1M 量子比特面临巨大工程挑战
4. **低温需求**：超导量子比特需要 mK 级低温
5. **算法限制**：尚未找到"杀手级"应用

---

## 4. 综合对比

| 技术 | 理论基础 | 成熟度 | 适合应用 | 挑战 |
|------|---------|-------|---------|------|
| 光计算 | 光学干涉 | 早期原型 | 矩阵乘法 | 精度、集成、非线性 |
| 模拟计算 | 模拟电路 | 小规模商用 | AI 推理 | 精度、噪声、扩展 |
| 量子计算 | 量子力学 | NISQ (~1000 qubit) | 分解、搜索、化学 | 纠错、温度、扩展 |
| FPGA | 数字逻辑 | 成熟 | 低延迟、信号处理 | 频率低、编程难 |
| GPU | SIMT | 非常成熟 | 大规模并行 | 功耗高、延迟大 |

**结论**：短期内（5-10 年），GPU/FPGA 仍占主导。光计算和模拟计算可能率先在特定 AI 推理场景中找到商业价值。量子计算需要突破纠错阈值才能展示有实际意义的量子优势。
