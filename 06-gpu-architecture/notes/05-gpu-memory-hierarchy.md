# 05 — GPU 内存层次

> GPU 拥有复杂的内存层次结构，理解每层存储的特性是优化 CUDA 程序性能的关键。

---

## 1. GPU 内存层次概览

GPU 内存层次从快到慢、从私有到共享：

```
                每个线程私有
┌─────────────────────────────────────────────────────────┐
│  寄存器 (Register)                                       │
│  速度: 0.5-1 周期   容量: ~256 个 32-bit 寄存器/线程      │
└─────────────────────────────────────────────────────────┘
                         │
                每个线程块共享
┌─────────────────────────────────────────────────────────┐
│  共享内存 (Shared Memory)                                │
│  速度: ~1-2 周期  容量: 48-164 KB / SM (取决于架构)      │
└─────────────────────────────────────────────────────────┘
                         │
                         │  每个 SM 私有
┌─────────────────────────────────────────────────────────┐
│  L1 缓存 (L1 Cache)                                     │
│  速度: ~1-2 周期  容量: 32-128 KB / SM                  │
└─────────────────────────────────────────────────────────┘
                         │
                所有 SM 共享
┌─────────────────────────────────────────────────────────┐
│  L2 缓存 (L2 Cache)                                     │
│  速度: ~30-50 周期  容量: 6-50 MB                       │
└─────────────────────────────────────────────────────────┘
                         │
                所有 SM 共享 (显存)
┌─────────────────────────────────────────────────────────┐
│  全局内存 / HBM (Global Memory)                          │
│  速度: ~400-800 周期  容量: 32-80 GB                    │
│  带宽: 900-3350 GB/s                                    │
└─────────────────────────────────────────────────────────┘
                         │
                特殊的只读存储
┌─────────────────────────────────────────────────────────┐
│  常量内存 (Constant Memory)  | 纹理内存 (Texture Mem)   │
│  ~64 KB, 广播式读取         |  大容量, 空间优化缓存     │
└─────────────────────────────────────────────────────────┘
```

### 关键性能参数对比

| 存储层级 | 延迟（周期） | 延迟（纳秒, 1.8GHz） | 容量 | 带宽 | 作用域 |
|---------|-------------|-------------------|------|------|--------|
| 寄存器 | 0.5-1 | ~0.3-0.6 ns | 256/线程 | 极高（~20 TB/s） | 线程私有 |
| 共享内存 | 1-2 | ~0.6-1.1 ns | 48-228 KB/SM | 极高 | 线程块 |
| L1 缓存 | 1-2 | ~0.6-1.1 ns | 32-128 KB/SM | 极高 | SM |
| L2 缓存 | 30-50 | ~17-28 ns | 6-50 MB | 高 | GPU |
| 全局内存 | 400-800 | ~220-440 ns | 32-80 GB | 3350 GB/s (H100) | GPU |
| 常量内存 | 400-600 | ~220-330 ns | 64 KB | 广播 | GPU |

---

## 2. 寄存器（Register）

### 特性

- 最快的存储，SM 内部最多的存储资源
- 每个线程私有，其他线程不可见
- 编译器和硬件分配，程序员不能直接声明（编译器自动分配）
- Fermi 架构后支持 32-bit 寄存器，双精度/64-bit 操作占用 2 个寄存器

### 容量限制

| 架构 | 寄存器/SM | 最大寄存器/线程 |
|------|-----------|---------------|
| Fermi | 32,768 (128 KB) | 63 |
| Kepler | 65,536 (256 KB) | 255 |
| Maxwell | 65,536 (256 KB) | 255 |
| Pascal | 65,536 (256 KB) | 255 |
| Volta | 65,536 (256 KB) | 255 |
| Turing | 65,536 (256 KB) | 255 |
| Ampere | 65,536 (256 KB) | 255 |
| Hopper | 65,536 (256 KB) | 255 |

### 寄存器溢出（Register Spilling）

当单个线程需要超过最大寄存器数量的变量时，多余的变量会被"溢出"到本地内存（在显存中）：

```cuda
// 编译器会根据代码复杂度分配寄存器
// 可以使用 --maxrregcount 限制每个线程的寄存器数
// 使用 ptxas --verbose 查看寄存器使用情况

__global__ void kernel() {
    // 声明大量变量可能导致寄存器溢出
    float a, b, c, d, e, f, g, h, ...;
    // 如果寄存器不够，部分变量存入本地内存（L1/显存）
}
```

**寄存器溢出 vs 手动控制寄存器**：
- 溢出性能损失大：本地内存访问延迟 > 寄存器
- 限制 `maxrregcount` 可以减少每个线程的寄存器使用，提高占用率
- 但可能因为溢出导致性能反而下降——需要权衡

---

## 3. 共享内存（Shared Memory）

### 特性

- 片上 SRAM，访问延迟 ~1-2 个周期
- 同一线程块内的所有线程共享
- 程序员显式管理（声明 `__shared__` 变量）
- 可与 L1 缓存配置动态分割

### 声明和使用

```cuda
// 静态声明
__shared__ float s_data[256];

// 动态声明（在 kernel launch 时指定大小，第三个参数）
__global__ void kernel() {
    extern __shared__ float s_data[];
}

// 调用时指定动态共享内存大小
kernel<<<grid, block, 256 * sizeof(float)>>>();
```

### 共享内存大小（每 SM）

| 架构 | 共享内存最大值 | L1+共享内存总和 | 配置方式 |
|------|---------------|----------------|---------|
| Fermi | 48 KB | 64 KB | 16/48 或 48/16 分割 |
| Kepler | 48 KB | 64 KB | 16/48 或 48/16 |
| Maxwell | 64 KB | 96 KB | 0/64 或 32/64 |
| Pascal | 64 KB | 96 KB | 0/64 或 32/64 |
| Volta | 96 KB | 128 KB | 三级可配置 |
| Turing | 64 KB | 96 KB | 0/64 或 32/64 |
| Ampere | 164 KB | 192 KB | 可编程 |
| Hopper | 228 KB | 256 KB | 可编程 |

---

## 4. L1 缓存

### 特性

- 片上缓存，延迟 ~1-2 周期
- 每个 SM 独立拥有
- 从 Fermi 到 Maxwell：与共享内存统一物理存储，可配置分割
- 从 Volta 开始：L1 和共享内存物理分离（但仍可配置）

### L1 缓存策略

```cuda
// 编译时控制 L1 缓存策略
__global__ void kernel() {
    // 使用 ldg() 强制通过 L1/texture 缓存加载
    float val = __ldg(&data[threadIdx.x]);
    
    // 默认行为取决于编译器优化
}
```

**L1 缓存的潜在问题**：
- 对于某些访问模式，L1 可能因为缓存争用（cache thrashing）而降低性能
- 使用 `__ldg()` 强制只读路径可以绕过 L1

---

## 5. L2 缓存

### 特性

- 所有 SM 共享的最后一级缓存
- 所有全局内存和纹理内存的访问都要经过 L2
- 缓存块大小为 32 字节（某些架构）或 128 字节
- 用于减少 HBM 带宽压力

### L2 分块

现代 GPU 的 L2 被分为多个独立分区，每个分区连接到一个 HBM 控制器：

```
GPU 内存控制器布局：
┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐
│ L2 0 │ │ L2 1 │ │ L2 2 │ │ L2 3 │  ... 共 24-40 个分区
├──────┤ ├──────┤ ├──────┤ ├──────┤
│ HBM0 │ │ HBM1 │ │ HBM2 │ │ HBM3 │  ... 每组连接一个 HBM 堆栈
└──────┘ └──────┘ └──────┘ └──────┘
```

### 容量的影响

更大的 L2 缓存可以显著减少对 HBM 的访问：
- V100: 6 MB → 不适合大工作集
- A100: 40 MB → 可容纳许多深度学习模型的一部分激活值
- H100: 50 MB → 可容纳更大的工作集

---

## 6. 全局内存（Global Memory）

### 特性

- GPU 中最大的存储空间，位于片外 HBM 或 GDDR
- 所有线程可读写
- 所有 SM 通过 L2 缓存访问
- 延迟 ~400-800 周期，**是 GPU 性能的主要瓶颈**

### 显存技术对比

| 显存类型 | 典型带宽 | 总线宽度 | 典型容量 | 延迟 |
|---------|---------|---------|---------|------|
| GDDR6 | ~600 GB/s (RTX 4090) | 384-bit | 24 GB | ~300-500 ns |
| GDDR6X | ~1000 GB/s (RTX 4090) | 384-bit | 24 GB | ~300-500 ns |
| HBM2 | ~900 GB/s (V100) | 4096-bit | 16-32 GB | ~200-400 ns |
| HBM2e | ~2.0 TB/s (A100) | 5120-bit | 40-80 GB | ~200-400 ns |
| HBM3 | ~3.35 TB/s (H100) | 5120-bit | 80 GB | ~200-400 ns |

### 本地内存（Local Memory）

当寄存器不足时，编译器将变量溢出到本地内存。本地内存本质上是全局内存的一部分，但每个线程使用自己的地址范围。性能影响：
- 访问本地内存 = 全局内存延迟（~400-800 周期）
- 与寄存器访问相比，性能差异巨大

---

## 7. 常量内存（Constant Memory）

### 特性

- 只读，大小为 64 KB
- 有专门的恒定缓存（constant cache），每个 SM 独立
- **广播**：warp 内所有线程读取同一地址时，仅一次访存后广播到所有线程
- 如果 warp 内线程读取不同地址，访问被串行化（一个地址一个周期）

### 使用场景

适合存储所有线程都访问相同地址的查找表、系数和常量数组：

```cuda
// 声明常量内存
__constant__ float coef[256];

__global__ void kernel() {
    // 所有线程读取同一地址 → 广播，高效
    float c = coef[0];
    
    // warp 内线程读取不同地址 → 串行化，低效
    float d = coef[threadIdx.x];
}
```

---

## 8. 纹理内存（Texture Memory）

### 特性

- 只读，通过纹理缓存（texture cache）访问
- **空间局部性优化**：利用 2D/3D 空间邻近性的缓存策略
- 支持硬件插值（线性滤波）
- 支持边界处理（clamp, wrap, mirror）
- 适用于图像处理和非对齐访问

### 与全局内存对比

| 对比维度 | 纹理内存 | 全局内存 |
|---------|---------|---------|
| 缓存策略 | 空间局部性优化 | 一般（L1/L2） |
| 只读 | 是 | 可读写 |
| 延迟 | L1 命中时 ~1-2 周期 | ~400-800 周期 |
| 适用场景 | 图像处理、不规则访问 | 通用数据 |
| 编程复杂度 | 需要纹理对象/引用 | 直接指针访问 |

## 9. 内存层次优化原则

### 原则 1：尽可能靠近计算

数据用得越频繁，存放位置应越快。应该从寄存器开始，依次考虑共享内存、L1/L2 缓存，最后才使用全局内存。

### 原则 2：最大化空间局部性

线程块内线程应访问连续内存地址，以利用缓存和合并访问。

### 原则 3：减少全局内存访问

每次全局内存访问都尽可能复用数据。将数据加载到共享内存后，多个线程重复使用。

### 原则 4：平衡占用率和寄存器使用

```
高寄存器使用 → 低占用率 → 低延迟隐藏能力
低寄存器使用 → 高占用率 → 更多溢出 → 性能下降
```

最佳点需要针对特定 kernel 和 GPU 架构调整。
