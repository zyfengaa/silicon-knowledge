# 05 — 存储层次

> CPU 越来越快，内存跟不上——存储层次是弥补这个速度差距的核心手段。理解为什么你的程序"明明很简单却跑得慢"。

---

## 本模块内容

| 笔记 | 主题 | 核心问题 |
|------|------|---------|
| 01 | **为什么需要存储层次** | 速度、容量、成本的不可能三角 |
| 02 | **缓存基础** | 命中/缺失、时间局部性、空间局部性 |
| 03 | **缓存组织方式** | 直接映射、组相联、全相联：硬件实现与命中率的取舍 |
| 04 | **缓存策略** | 替换策略（LRU/伪LRU/随机）、写策略（Write-through vs Write-back） |
| 05 | **TLB** | 快表：加速虚拟地址到物理地址转换 |
| 06 | **虚拟内存** | 页表、多级页表、大页、缺页中断 |
| 07 | **DRAM 基础** | 行/列地址结构、刷新周期、预充电、DDR 演化 |
| 08 | **HBM 与 GDDR** | 高带宽内存方案对比：DDR5 / GDDR6/7 / HBM2e/3 |

## 建议学习方式

1. 先理解"为什么需要缓存"——用一条 `ld` 指令的延迟对比说明
2. 逐层深入：寄存器 → L1 → L2 → L3 → DRAM → SSD
3. 运行缓存模拟器，尝试不同的参数配置观察命中率变化
4. 运行矩阵乘分块示例，观察分块大小对性能的影响
5. 用 `perf stat` 分析真实程序的缓存表现
