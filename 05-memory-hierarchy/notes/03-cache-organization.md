# 05-03 缓存组织方式

## 缓存地址的三个字段

CPU 发出的内存地址在被缓存控制器解析时，被拆分为三个字段：

```
┌────────────────────┬──────────────┬──────────────┐
│       Tag          │    Index     │    Offset    │
└────────────────────┴──────────────┴──────────────┘
```

- **Offset（偏移量）**：确定数据在缓存行内的位置。位数 = log₂(缓存行大小)
- **Index（索引）**：确定该地址映射到缓存的哪一组（set）。位数取决于缓存的组数
- **Tag（标记）**：用于区分映射到同一索引的不同内存地址。位数 = 地址总位数 - Index 位数 - Offset 位数

## 三种缓存组织方式

### 1. 直接映射缓存（Direct-Mapped Cache）

每个内存地址只能映射到**唯一一个**缓存行位置。

```
索引 = (地址 / 行大小) % 缓存行数
```

**结构示意**：

```
索引   Tag  数据    有效位
0     ┌─────┬──────┬───┐
1     ├─────┼──────┼───┤
2     ├─────┼──────┼───┤
...   ├─────┼──────┼───┤
N-1   └─────┴──────┴───┘
```

**优点**：硬件简单，访问速度快（先读 Tag，同时开始读数据，比较后选择）。
**缺点**：容易产生冲突缺失——如果两个频繁访问的地址映射到同一行，会互相驱逐。

**示例**：64 KB 缓存，128 行，行大小 512 B
```
Offset bits = log₂(64) = 6         // 64 字节缓存行
Index bits  = log₂(1024) = 10      // 1024 组
Tag bits    = 64 - 10 - 6 = 48     // 64 位地址
```

### 2. 全相联缓存（Fully-Associative Cache）

任何内存地址可以放在任何缓存行位置。

**优点**：没有冲突缺失，缓存利用率最高。
**缺点**：每次访问需要将 Tag 与**所有**缓存行的 Tag 同时比较——硬件代价极高。不适用于大型缓存。

通常只用于 TLB 等小型缓存。

**结构示意**：

```
行号   Tag  数据    有效位
0     ┌─────┬──────┬───┐
1     ├─────┼──────┼───┤
2     ├─────┼──────┼───┤
...   ├─────┼──────┼───┤
N-1   └─────┴──────┴───┘
```

**注意**：没有 Index 字段——需要全量 Tag 比较。

### 3. 组相联缓存（Set-Associative Cache）

N 路组相联是直接映射和全相联的折中：缓存被分为若干组，每组包含 N 行。每个地址映射到**一组**，组内可以放在**任意一行**。

```
组号 = (地址 / 行大小) % 组数
```

**结构示意（2 路组相联）**：

```
组   路 0              路 1
0    ┌──────┬──────┬───┐ ┌──────┬──────┬───┐
     │ Tag  │ 数据  │ V │ │ Tag  │ 数据  │ V │
1    ├──────┼──────┼───┤ ├──────┼──────┼───┤
...  ├──────┼──────┼───┤ ├──────┼──────┼───┤
M-1  └──────┴──────┴───┘ └──────┴──────┴───┘
```

**优点**：冲突缺失显著少于直接映射，硬件复杂度低于全相联。
**缺点**：访问时间随路数增加而增长（需要并行比较多个 Tag）。

## 地址字段计算

以 **64 KB 缓存、64 B 缓存行、4 路组相联** 为例：

**Step 1: Offset**
```
Offset bits = log₂(64) = 6
```

**Step 2: 组数**
```
每组 4 行
缓存总行数 = 64 KB / 64 B = 1024 行
组数 = 1024 / 4 = 256 组
```

**Step 3: Index**
```
Index bits = log₂(256) = 8
```

**Step 4: Tag**
```
地址宽度 = 64 位（x86-64）
Tag bits = 64 - 8 (Index) - 6 (Offset) = 50
```

**Step 5: 地址分解示例**
```
地址 0x00007F00_00001234 (二进制省略):
  Tag:     0x00007F00_0000       48 bits  ← 高位
  Index:   0x48                   8 bits   ← 中间位
  Offset:  0x34                   6 bits   ← 低位
```

## 关联度权衡

| 关联度 | 冲突缺失 | 访问延迟 | 硬件复杂度 |
|--------|---------|---------|-----------|
| 直接映射 (1-way) | 高 | 快 | 低 |
| 2-way | 中 | 较快 | 中 |
| 4-way | 低 | 一般 | 较高 |
| 8-way | 更低 | 较慢 | 高 |
| 全相联 | 无 | 最慢 | 极高 |

现代 CPU 的权衡：

- **L1 缓存**：通常 4–8 路，需要在速度和命中率之间平衡
- **L2 缓存**：通常 8–12 路，容量较大时可以容纳更多路
- **L3 缓存**：通常 12–20 路，容量最大

**关联度的收益递减**：从 1-way 到 2-way 的冲突缺失改善最显著；从 4-way 到 8-way 改善较小；超过 8-way 后收益极微。

## 关键概念

- **地址分解**：地址分为 Tag / Index / Offset 三个字段
- **直接映射**：每个地址映射到固定位置，简单但冲突多
- **全相联**：灵活性最高，但硬件复杂，只适合小容量
- **组相联**：最佳折中——现代 CPU 的标准选择
- **关联度**：通过牺牲少量时间换取大幅降低冲突缺失
