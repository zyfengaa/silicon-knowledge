# Module 02: ISA and CPU Basics
## 05 -- Control Unit Design

### Purpose of the Control Unit

The control unit generates the control signals that direct the datapath. For each instruction, it must set the correct values for signals such as:

| Control Signal    | Function                                                     |
|-------------------|--------------------------------------------------------------|
| RegWrite          | Enable write to register file (1 = write)                    |
| ALUSrc            | Select ALU B input (0 = register, 1 = immediate)            |
| MemWrite          | Enable write to data memory (1 = store)                      |
| MemRead           | Enable read from data memory (1 = load)                      |
| MemtoReg          | Select write-back data (0 = ALU result, 1 = memory data)    |
| ALUOp             | ALU operation code (2 bits encoding the operation class)     |
| Branch            | Enable branch target selection (1 = branch instruction)      |
| Jump              | Enable jump target selection (1 = jump instruction)          |
| PCSrc             | Select next PC (0 = PC+4, 1 = branch/jump target)           |

### Hardwired vs. Microprogrammed Control

| Aspect               | Hardwired Control                               | Microprogrammed Control                    |
|----------------------|-------------------------------------------------|--------------------------------------------|
| Implementation       | Combinational logic gates                       | ROM + microsequencer                       |
| Speed                | Fast (critical path = gate delays)              | Slower (ROM access + sequencing)           |
| Flexibility          | Difficult to modify (redesign logic)            | Easy to modify (change microcode)          |
| Design complexity    | High for complex ISAs                           | Lower for complex ISAs                     |
| Used in              | RISC CPUs, simple pipelines                     | CISC CPUs (pre-1990s x86, VAX, 68000)     |
| Example              | RISC-V single-cycle, MIPS, ARM Cortex-M         | Intel 486, VAX-11/780                      |

RISC-V typically uses hardwired control because the ISA is simple and uniform. Modern x86 CPUs decode CISC instructions into micro-ops at runtime, which is a hybrid approach.

### Control Signal Generation

For a single-cycle RISC-V CPU, control signals are generated by:
1. Decoding the **opcode** bits [6:0] to identify the instruction class
2. Using **funct3** and **funct7** (for R-type) to refine the ALU operation
3. Combining these into logic equations or a truth table

The following truth table shows control signal values for each instruction class. "X" = don't care.

#### Signal Definitions

**RegWrite**: 1 if the instruction writes to the register file.
- R-type, I-type ALU, LW, LUI, AUIPC, JAL, JALR = 1
- SW, branches, JALR (if rd=x0) = 0

**ALUSrc**: Selects the second ALU operand.
- 0 = Read data 2 from register file (register operand)
- 1 = Immediate value (from instruction encoding)
- R-type, branches = 0; I-type, LW, SW = 1

**MemWrite**: Write enable for data memory. Only set for store instructions.

**MemRead**: Read enable for data memory. Only set for load instructions.

**MemtoReg**: Selects what gets written to the register file.
- 0 = ALU result
- 1 = Data memory output (for loads only)

**Branch**: 1 for branch instructions (BEQ, BNE, etc.). The ALU computes the comparison; the result ANDed with Branch generates the actual PCSrc signal for taken branches.

**Jump**: 1 for JAL and JALR. Forces PCSrc = 1 independent of ALU comparison.

**ALUOp**: Two-bit code that selects how ALU control generates the specific ALU operation:
- 00 = Load/store (ADD, for address calculation)
- 01 = Branch (SUB, for register comparison)
- 10 = R-type/I-type (decode from funct3/funct7)

### ALU Control Logic

The ALU control block takes ALUOp, funct3, funct7, and opcode to produce the exact ALU operation.

| ALUOp | Instruction Class | funct3 | funct7[5] | ALU Operation | ALU Control |
|-------|-------------------|--------|-----------|---------------|-------------|
| 00    | LW/SW            | X      | X         | ADD           | 0000 (ADD)  |
| 01    | Branch           | X      | X         | SUB (compare) | 0001 (SUB)  |
| 10    | R-type           | 000    | 0         | ADD           | 0000        |
| 10    | R-type           | 000    | 1         | SUB           | 0001        |
| 10    | R-type           | 100    | 0         | XOR           | 0100        |
| 10    | R-type           | 110    | 0         | OR            | 0110        |
| 10    | R-type           | 111    | 0         | AND           | 0111        |
| 10    | R-type           | 001    | 0         | SLL           | 1000        |
| 10    | R-type           | 101    | 0         | SRL           | 1001        |
| 10    | R-type           | 101    | 1         | SRA           | 1010        |
| 10    | I-type           | 000    | X         | ADD           | 0000        |
| 10    | I-type           | 110    | X         | OR            | 0110        |

### PCSrc Generation

The final PC source select is:

```
PCSrc = Branch & (ALU_Zero XOR invert_branch_condition) | Jump
```

Where:
- For BEQ: Branch=1, ALU does SUB; if zero=1, branch taken
- For BNE: Branch=1, invert=1; branch taken if zero=0
- For JAL/JALR: Jump=1, unconditionally select jump target

### Logic Equations for Control Signals

In a hardware description language (Verilog/VHDL), the control unit can be expressed as combinational logic:

```verilog
always_comb begin
    // Defaults
    RegWrite = 0;  ALUSrc = 0;  MemWrite = 0;
    MemRead = 0;   MemtoReg = 0; Branch = 0;
    Jump = 0;      ALUOp = 2'b00;

    case (opcode)
        7'b0110011: begin              // R-type
            RegWrite = 1;  ALUOp = 2'b10;
        end
        7'b0010011: begin              // I-type ALU
            RegWrite = 1;  ALUSrc = 1;  ALUOp = 2'b10;
        end
        7'b0000011: begin              // LW
            RegWrite = 1;  ALUSrc = 1;  MemRead = 1;
            MemtoReg = 1;  ALUOp = 2'b00;
        end
        7'b0100011: begin              // SW
            ALUSrc = 1;  MemWrite = 1;  ALUOp = 2'b00;
        end
        7'b1100011: begin              // Branch
            Branch = 1;  ALUOp = 2'b01;
        end
        7'b1101111: begin              // JAL
            RegWrite = 1;  Jump = 1;
        end
        7'b1100111: begin              // JALR
            RegWrite = 1;  ALUSrc = 1;  Jump = 1;  ALUOp = 2'b00;
        end
        7'b0110111: begin              // LUI
            RegWrite = 1;  ALUSrc = 1;  ALUOp = 2'b10;
        end
        7'b0010111: begin              // AUIPC
            RegWrite = 1;  ALUSrc = 1;  ALUOp = 2'b10;
        end
    endcase
end
```

```

|   (opcode)  |     |                        |----> MemWrite
|             |     |                        |----> MemRead
|   [14:12]   |     |                        |----> ALUSrc
|   (funct3)  |     |                        |----> MemtoReg
|             |     |                        |----> Branch
|   [31:25]   |     |                        |----> Jump
|   (funct7)  |     |                        |----> ALUOp[1:0]
|             |     +------+-----------------+
+-------------+            |
                           | ALUOp, funct3, funct7
                           v
                    +--------------+
                    | ALU Decoder  |----> ALU control[3:0]
                    +--------------+
```

### Summary

- The control unit translates instruction bits into datapath control signals
- Hardwired control = fast, simple, good for RISC ISAs
- Microprogrammed control = flexible, complex, useful for CISC ISAs
- Control signals are derived from opcode, funct3, and funct7 fields
- ALUOp is a 2-bit intermediate that simplifies the ALU control decoder
- The truth table for a single-cycle RISC-V control unit is relatively small (9 instruction classes)
