# 02 — Amdahl's Law 与 Gustafson's Law

> 并行加速有两个经典的数学模型。Amdahl 定律告诉我们"在固定问题规模下，加速是有上限的"；Gustafson 定律则指出"当问题规模随处理器数量增长时，加速也可以持续增长"。两者描述的是**强扩展（strong scaling）** 和**弱扩展（weak scaling）** 两种不同的视角。

---

## 1. Amdahl's Law (1967)

### 1.1 基本公式

Gene Amdahl 在 1967 年提出了并行加速的基本限制：

```
加速比 S(N) = 1 / ((1 - P) + P / N)

其中：
P = 程序中可以并行的比例 (0 ≤ P ≤ 1)
(1-P) = 必须串行执行的比例
N = 处理器核心数
```

### 1.2 直观理解

```
假设一个程序:
- 90% 可以并行 (P = 0.9)
- 10% 必须串行 (1 - P = 0.1)

N = 2:    S = 1 / (0.1 + 0.9/2) = 1 / 0.55 ≈ 1.82×
N = 4:    S = 1 / (0.1 + 0.9/4) = 1 / 0.325 ≈ 3.08×
N = 8:    S = 1 / (0.1 + 0.9/8) = 1 / 0.2125 ≈ 4.71×
N = 16:   S = 1 / (0.1 + 0.9/16) = 1 / 0.15625 ≈ 6.40×
N → ∞:    S_max = 1 / 0.1 = 10×
```

即使有无穷多个核心，92.5% 并行的程序最高也只能加速 13.3×。

### 1.3 关键洞察

```
Amdahl 的核心结论:
S_max = 1 / (1 - P)   当 N → ∞

即使 99% 的代码已经并行化:
S_max = 1 / (1 - 0.99) = 100×

那 1% 的串行部分决定了加速的上限。
```

### 1.4 实际影响

| 并行比例 P | 最大加速比 (N→∞) | 16 核加速比 | 64 核加速比 |
|-----------|------------------|------------|------------|
| 0.5 (50%) | 2× | 1.86× | 1.97× |
| 0.9 (90%) | 10× | 6.40× | 8.42× |
| 0.99 (99%) | 100× | 13.9× | 39.3× |
| 0.999 (99.9%) | 1000× | 15.5× | 61.5× |

**观察**：当 P=0.99 时，64 核心才能获得 39× 加速——低于 64 核心的 61% 效率。

## 2. Gustafson's Law (1988)

### 2.1 动机

Gustafson 指出 Amdahl 定律有一个隐含假设：**问题规模固定**。

但在实际中，当你有更多处理器时，你通常会**处理更大的问题**而不是追求更快完成同一问题。

```
Amdahl: "同样的计算，1 小时完成 vs. N 核上 X 分钟完成"
Gustafson: "单核 1 小时能做的计算 vs. N 核 1 小时能做的计算"
```

### 2.2 基本公式

Gustafson's Law (也称 scaled speedup):

```
加速比 S(N) = N - α(N - 1)

其中：
α = 程序中的串行部分比例 (相对于扩展后的问题规模)
N = 处理器核心数

更常见的推导:
S(N) = 1 × (1 - P') + N × P' = 1 - P' + N × P' = 1 + P'(N - 1) = N - α(N - 1)

其中 P' 是基于扩展后问题规模的并行比例。
```

### 2.3 对比示例

```
Amdahl 视角 (固定问题规模):
- 串行时间: 10 分钟
- 并行时间: 90 分钟
- 总时间: 100 分钟
- 10 核加速比: 100 / (10 + 90/10) = 100 / 19 = 5.26×

Gustafson 视角 (固定执行时间):
- 串行部分仍然是 10 分钟 (问题规模扩大后)
- 并行部分可以在 10 核上完成 900 分钟的工作
- 总工作量: 10 + 900 = 910 分钟
- 加速比: 910 / 100 = 9.1×
```

## 3. 强扩展与弱扩展

### 3.1 定义

| 概念 | 描述 | 对应定律 |
|------|------|---------|
| **强扩展 (Strong Scaling)** | 总问题规模不变，增加处理器数量，看执行时间如何减少 | Amdahl's Law |
| **弱扩展 (Weak Scaling)** | 每个处理器的负载不变，增加处理器时问题规模同步增加 | Gustafson's Law |

### 3.2 强扩展效率

```
强扩展效率 = S(N) / N × 100%

N = 64, P = 0.99:
S = 1 / (0.01 + 0.99/64) = 39.3×
效率 = 39.3 / 64 = 61.4%
```

当效率低于 50% 时，再增加核心数的性价比骤降。

### 3.3 弱扩展效率

```
弱扩展效率 = S(N) / N × 100%

根据 Gustafson:
α = 0.01 (串行 1%)
S(N) = N - 0.01(N - 1) = 0.99N + 0.01
效率 = (0.99N + 0.01) / N ≈ 99%
```

弱扩展效率通常很高，因为串行部分占比随问题规模增大而减小。

## 4. 实际案例

### 4.1 深度学习训练

```
GPT-3 训练在 10,000 GPU 集群上:

强扩展问题:
- 一个训练步: 串行部分包括梯度 All-Reduce 的等待时间
- 随着 GPU 数增加，一个 batch 的计算时间减少
- 但通信开销降低缓慢 (受限于网络带宽)
- P 逐渐减小

实际公式 (考虑通信开销):
S(N) = 1 / (1 - P + P/N + C(N))

C(N) = 通信开销，通常随 N 增长

在 10K GPU 规模:
- 计算时间占 ~30%
- 通信时间占 ~40%
- 空闲等待占 ~30%
```

### 4.2 科学计算

```
分子动力学模拟 (NAMD):

弱扩展测试 (每个核心固定 ~10,000 原子):
N=64:   99% 效率
N=512:  95% 效率
N=4096: 88% 效率

随着核心数增加，边界通信开销逐渐增大，效率缓慢下降。
```

## 5. 实际应用指南

### 5.1 如何衡量 P

```
方法 1: 分析测量
- 编写一个串行版本 → 测量总时间 T_serial
- 编写一个完美并行版本 → 测量并行时间 T_parallel
- P = T_parallel / (T_parallel + T_serial)

方法 2: 多核测量
- 分别在 1 核、2 核、4 核、8 核上运行
- 观察加速比曲线 → 拟合得到 P

方法 3: Amdahl 图
- 绘制 1/S vs. 1/N → 截距 = (1-P)
```

### 5.2 何时使用哪个定律

| 场景 | 应该使用 |
|------|---------|
| "我想加快这个固定大小的计算" | Amdahl (强扩展) |
| "我想用更多 GPU 训练更大的模型" | Gustafson (弱扩展) |
| "我有 1 万个数据点，要分到 100 个核上" | 弱扩展 (每个核处理 100 个) |
| "我有 1 万个数据点，希望 100 个核越快越好" | 强扩展 |
